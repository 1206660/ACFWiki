    Battle Zone

    ACFBattleZone

    ACFBattleZone is the actor you place in the level to define where a turn-based battle takes place. It is a Pawn — the player controller possesses it during combat so the battle camera becomes the active view. All configuration is on the ACFBattleZoneComponent it owns.

    ACFBattleZoneComponent

    The core component on ACFBattleZone. It holds spawn slots for heroes and enemies, manages the battle camera, spawns enemies at battle start, and cleans up at battle end.

    Spawn positions (ACF | Positions)

    • HeroesPositions — array of FTransform defining where party members are placed. Supports viewport widget gizmos: drag the handles directly in the editor.

    • EnemyPositions — same for enemies. The number of slots here is the hard cap on how many enemies can appear; extras from the encounter are silently skipped.

    • Both arrays draw wire capsules in the editor (green = heroes, red = enemies) for easy visual placement.

    • If both arrays are left empty, 3 default slots are auto-generated for each side.



    Debug visualization (ACF | Debug)

    • bShowDebugCapsules — draws wire capsules at every slot in the editor viewport (default true, no runtime cost).

    • DebugCapsuleRadius / DebugCapsuleHalfHeight — adjust to match your character's capsule size.

    What happens at battle start

    When StartBattle(EncounterDataAsset, PlayerController) is called:

    1. The overworld player pawn is hidden, collision disabled, movement stopped.

    2. The player controller possesses the ACFBattleZone pawn — battle camera activates.

    3. Active party members are teleported to hero slots (ground-snapped if needed) and made visible.

    4. Enemies are spawned at enemy slots from the encounter data, stats initialized from their ACFCharacterDataAsset.

    5. Control is handed to ACFBattleManagerComponent on the GameState to run the turn logic.

    When EndBattle is called:

    • Spawned enemies are destroyed (equipment hidden before destruction to avoid visual pop).

    • Party members are hidden and frozen again.

    • Battle music is stopped.

    • The Battle Manager separately restores the player pawn and blends the camera back.

    Blueprint-callable functions

    • GetPartyMembers — returns party characters currently in this zone.

    • GetEnemies — returns spawned enemy pawns.

    • GetOriginTransformForPawn(Pawn) — returns the slot transform assigned to a pawn; abilities use this to know where a character should return after approaching a target.

    • ClearBattle — force-resets the zone state without going through the normal end flow.

    ACFEncounterDataAsset

    A PrimaryDataAsset describing everything about one encounter. Create one per encounter type in the Content Browser. Right-click on content browser-> Miscellaneous -> Data Asset and select the encounter Data Asset

    • Enemies — array of FACFEnemyGroup. Each entry has:

      • BaseUnit — the enemy character Blueprint class, starting level, and optional ACFCharacterDataAsset for stats and appearance.

      • Count — how many instances to spawn from this entry.

      • Entries fill enemy slots in order.

    • BattleMusic — SoundCue played during this encounter.

    • Rewards:

      • PartyExp — XP granted to active party on victory.

      • ReserveExp — XP granted to reserve characters.

      • Currency — currency dropped.

      • Items — array of item drops.

    • BattleStartSequences — LevelSequence assets played as intro cinematics.

    • BattleEndSequences — LevelSequence assets played as victory/defeat cinematics.

    ACFBattleZoneStarterComponent

    Add this to an overworld enemy actor to handle automatic battle triggering when the player gets close. It detects the player, resolves the Battle Zone, and calls StartBattle.


    Setup (ACF | BattleZone / ACF | Encounter)

    • BattleZone — soft reference to the ACFBattleZone actor in the level to use for this encounter.

    • Encounter — the ACFEncounterDataAsset describing the enemies to fight.


    Trigger (ACF | Trigger)

    • bAutoCreateTrigger (default true) — automatically creates a sphere trigger on the owner's root at BeginPlay.

    • TriggerRadius — radius of the auto-created sphere (default 200 units).

    • ExternalTrigger — assign your own collision component here if bAutoCreateTrigger is false (e.g. a box or existing capsule).

    • BindToTrigger(PrimitiveComponent) — bind a trigger at runtime from Blueprint.

    • bStartOnOverlap (default true) — set to false to disable automatic triggering and call StartBattle manually.

    On battle start (ACF | BattleZone)

    • bDestroyOwnerOnStart (default true) — destroys the overworld enemy actor after battle starts.

    • DestroyOwnerDelay — seconds to wait before destruction (default 0.8s), giving startup animations time to finish.

    • bApplyAppearanceFromEncounter — if true, loads the ACFCharacterDataAsset of the enemy at AppearanceEnemyIndex from the encounter and applies its mesh and materials to the owning actor. Useful to make a generic overworld enemy visually match the encounter without separate Blueprint classes per enemy type.
